# Markdown変換ツール（Word）設計書

## 1. 文書情報

| 項目 | 内容 |
| --- | --- |
| 文書名 | Markdown変換ツール（Word）設計書 |
| 対象リポジトリ | `md2doc` |
| 作成フェーズ | 設計 |
| 想定実装方式 | C# コンソールアプリ |
| 想定利用者 | 一般職員 |

## 2. ツール概要

本ツールは、Markdown形式のテキスト、またはMarkdownテキストファイルを入力として受け取り、
Microsoft Word形式（`.docx`）の文書を出力する。

追加要件として、出力文書全体に適用するフォント名とフォントサイズを、
利用者が指定できるようにする。

## 3. 目的

- 職員が作成したMarkdown文書を、配布・共有しやすいWord文書へ変換する。
- 変換時の文書スタイル（フォント名、フォントサイズ）を業務ルールに合わせる。
- 閉域環境でも利用可能な、外部通信不要の変換手段を提供する。

## 4. 対象利用者

- 一般職員（IT専門職でない利用者を含む）
- 定型文書をWord形式で提出・回覧する業務担当者

## 5. 利用環境・前提

- OS: Windows（閉域または庁内ネットワークを想定）
- 実行形式: C#コンソールアプリ（ローカル実行）
- 外部通信: 不要（外部API、外部CDN、クラウド連携は行わない）
- 追加要件: 外部依存を追加しない

## 6. スコープ

### 6.1 できること

- Markdown文字列入力を`.docx`へ変換する。
- Markdownファイル入力を`.docx`へ変換する。
- 出力先パスを指定して保存する。
- 出力文書のフォント名・フォントサイズを指定する。
- 変換成功・失敗をコンソールメッセージで通知する。

### 6.2 できないこと（今回の設計範囲外）

- PDFやHTMLなど、`.docx`以外の形式への出力
- 画像の自動ダウンロードや外部リンク先の取得
- GUI（画面アプリ）提供
- 個人情報を永続保存する機能
- クラウドストレージへの保存

## 7. 入力・処理・出力

### 7.1 入力

入力方法は以下2系統とする。

1. 文字列入力
   - コマンド引数または標準入力でMarkdown本文を受け取る。
2. ファイル入力
   - 入力ファイルパス（`.md` / `.txt`）を指定して読み込む。

追加指定項目:

- 出力ファイルパス（`.docx`）
- フォント名（例: `MS Gothic`）
- フォントサイズ（例: `11`）

### 7.2 処理

1. 入力パラメータ検証
   - 必須項目の有無
   - 入力ファイル存在確認
   - 出力先ディレクトリ存在確認
   - フォントサイズの数値範囲確認
2. Markdown解析
   - 見出し
   - 段落
   - 箇条書き（順序あり・なし）
   - 強調（太字・斜体）
   - コードブロック（等幅フォント扱い）
3. Word文書構築
   - 要素ごとに段落・書式を設定
   - 既定フォント（名称・サイズ）を全体適用
4. `.docx`出力
   - 指定パスへ保存
5. 結果表示
   - 成功時: 出力先を表示
   - 失敗時: 原因と対処ヒントを表示

### 7.3 出力

- Wordファイル（`.docx`）
- コンソールメッセージ（処理結果、エラー内容）

## 8. 想定コマンド仕様（案）

```text
md2doc --input "C:\input\sample.md" --output "C:\output\sample.docx" --font-name "MS Gothic" --font-size 11
```

```text
md2doc --text "# 見出し\n本文" --output "C:\output\from_text.docx" --font-name "Yu Gothic" --font-size 10.5
```

引数設計方針:

- `--input` と `--text` は排他指定とする。
- `--output` は必須とする。
- `--font-name`、`--font-size` は任意指定とし、未指定時は既定値を使用する。

## 9. フォント指定要件

- フォント名:
  - 利用者指定値を採用する。
  - 未指定時は既定値（例: `MS Gothic`）を適用する。
- フォントサイズ:
  - 利用者指定値を採用する。
  - 未指定時は既定値（例: `11`）を適用する。
  - 受入範囲（例: `8`〜`72`）外はエラーとする。

## 10. データ取り扱い方針

### 10.1 保存するデータ

- 利用者が指定した出力`.docx`ファイル
- 実行時に必要な一時データ（プロセス内メモリ）

### 10.2 保存しないデータ

- 個人情報を含む入力内容の永続ログ
- 外部サービスへの送信データ
- 変換元Markdownのバックアップ複製（明示要求がない限り）

## 11. セキュリティ・個人情報対応

- 外部送信を行わない（オフライン前提）。
- ログには入力本文を出力しない。
- エラーメッセージは機密情報を含めない。
- 一時ファイルを使用する実装にする場合は、保存先と削除手順を運用手順に明記する。

## 12. エラー時挙動

主なエラーケースと対応:

| エラーケース | 検知タイミング | 利用者向け表示 | 終了コード |
| --- | --- | --- | --- |
| 入力指定不足 | 起動時 | 必須引数不足を表示し使用例を案内 | 1 |
| 入力ファイルなし | 前処理 | ファイル不存在を表示 | 2 |
| 出力先不正 | 前処理 | 出力パス不正を表示 | 3 |
| フォントサイズ不正 | 前処理 | 許容範囲を表示 | 4 |
| 変換失敗 | 変換中 | 変換失敗と再確認項目を表示 | 5 |
| 保存失敗 | 保存時 | 書き込み不可理由を表示 | 6 |

## 13. 再実行時挙動

- 既存出力ファイルがある場合:
  - 既定は上書き禁止とし、確認フラグ（例: `--overwrite`）がある場合のみ上書きする。
- 同一入力で再実行した場合:
  - 入力・フォント指定が同一であれば、同一内容の`.docx`を再生成する。

## 14. 非機能要件

- 可用性:
  - 外部サービスに依存しない単体実行が可能。
- 保守性:
  - Markdown要素ごとの変換処理を関数分割する。
- 操作性:
  - 職員が短いコマンド例で実行できる。
- 監査性:
  - 実行結果（成功/失敗、出力先、終了コード）を簡潔に表示する。

## 15. テスト観点（設計段階）

- 正常系:
  - ファイル入力→docx出力成功
  - 文字列入力→docx出力成功
  - フォント名・サイズ指定反映
- 異常系:
  - 引数不足
  - 入力ファイル不存在
  - フォントサイズ範囲外
  - 出力先書き込み権限不足
- 境界値:
  - 最小/最大フォントサイズ
  - 空行や長文を含むMarkdown
- 再実行:
  - 上書き禁止時の挙動
  - 上書き許可時の挙動
