# Markdown変換ツール（Word）設計書

## 1. 文書情報

| 項目 | 内容 |
| --- | --- |
| 文書名 | Markdown変換ツール（Word）設計書 |
| 対象リポジトリ | `md2doc` |
| 作成フェーズ | 設計 |
| 想定実装方式 | C# WinFormsアプリ |
| 想定利用者 | 一般職員 |

## 2. ツール概要

本ツールは、Markdown形式のテキスト、またはMarkdownテキストファイルを入力として受け取り、
Microsoft Word形式（`.docx`）の文書を出力する。

追加要件として、出力文書全体に適用するフォント名とフォントサイズを、
利用者が画面上で指定できるようにする。

## 3. 目的

- 職員が作成したMarkdown文書を、配布・共有しやすいWord文書へ変換する。
- 変換時の文書スタイル（フォント名、フォントサイズ）を業務ルールに合わせる。
- 閉域環境でも利用可能な、外部通信不要の変換手段を提供する。

## 4. 対象利用者

- 一般職員（IT専門職でない利用者を含む）
- 定型文書をWord形式で提出・回覧する業務担当者

## 5. 利用環境・前提

- OS: Windows（閉域または庁内ネットワークを想定）
- 実行形式: C# WinFormsアプリ（ローカル実行）
- 利用端末前提: Microsoft Office Standard導入済み（Word利用可能）
- 外部通信: 不要（外部API、外部CDN、クラウド連携は行わない）
- 追加依存: 新規追加しない（端末導入済みのOffice機能範囲で実現）

## 6. スコープ

### 6.1 できること

- Markdown文字列入力を`.docx`へ変換する。
- Markdownファイル入力を`.docx`へ変換する。
- 出力先パスを指定して保存する。
- 出力文書のフォント名・フォントサイズを指定する。
- 変換成功・失敗を画面メッセージで通知する。

### 6.2 できないこと（今回の設計範囲外）

- PDFやHTMLなど、`.docx`以外の形式への出力
- 画像の自動ダウンロードや外部リンク先の取得
- クラウドストレージへの保存
- 個人情報を永続保存する機能

## 7. 画面設計（WinForms）

## 7.1 画面構成

単一画面で以下の領域を持つ。

1. 入力方式選択
   - ラジオボタン: `テキスト入力` / `ファイル入力`
2. 入力エリア
   - テキスト入力時: Markdown本文テキストボックス（複数行）
   - ファイル入力時: 入力ファイルパス + 参照ボタン
3. 出力設定エリア
   - 出力ファイルパス + 参照ボタン
   - フォント名入力（コンボボックスまたはテキストボックス）
   - フォントサイズ入力（数値入力）
4. 実行エリア
   - `変換実行` ボタン
5. 結果表示エリア
   - 成功/失敗メッセージ
   - エラー時の対処ヒント

## 7.2 UI/UX方針

- 専門用語を避け、ラベルは平易な日本語とする。
- 必須項目は明示し、未入力時は実行前に警告する。
- 成功時は保存先を表示し、次操作（ファイルを開く等）を案内する。
- エラー時は原因と対処を1〜2文で表示する。

## 8. 入力・処理・出力

## 8.1 入力

入力方法は以下2系統とする。

1. テキスト入力
   - 画面のMarkdown本文欄に直接貼り付ける。
2. ファイル入力
   - 入力ファイルパス（`.md` / `.txt`）を指定して読み込む。

追加指定項目:

- 出力ファイルパス（`.docx`）
- フォント名（例: `MS Gothic`）
- フォントサイズ（例: `11`）

## 8.2 処理

1. 入力値検証
   - 入力方式に応じた必須項目確認
   - 入力ファイル存在確認
   - 出力先ディレクトリ存在確認
   - フォントサイズ範囲確認
2. Markdown解析
   - 見出し
   - 段落
   - 箇条書き（順序あり・なし）
   - 強調（太字・斜体）
   - コードブロック（等幅フォント扱い）
3. Word文書生成
   - Microsoft Office Standard（Word）利用前提で`.docx`を生成
   - 要素ごとに段落・書式を設定
   - 指定フォント（名称・サイズ）を全体適用
4. `.docx`保存
   - 指定パスへ保存
5. 結果表示
   - 成功時: 出力先を表示
   - 失敗時: 原因と対処ヒントを表示

## 8.3 出力

- Wordファイル（`.docx`）
- 画面メッセージ（処理結果、エラー内容）

## 9. フォント指定要件

- フォント名:
  - 利用者指定値を採用する。
  - 未指定時は既定値（例: `MS Gothic`）を適用する。
- フォントサイズ:
  - 利用者指定値を採用する。
  - 未指定時は既定値（例: `11`）を適用する。
  - 受入範囲（例: `8`〜`72`）外はエラーとする。

## 10. データ取り扱い方針

## 10.1 保存するデータ

- 利用者が指定した出力`.docx`ファイル
- 実行時に必要な一時データ（プロセス内メモリ）

## 10.2 保存しないデータ

- 個人情報を含む入力内容の永続ログ
- 外部サービスへの送信データ
- 変換元Markdownのバックアップ複製（明示要求がない限り）

## 11. セキュリティ・個人情報対応

- 外部送信を行わない（オフライン前提）。
- ログには入力本文を出力しない。
- エラーメッセージは機密情報を含めない。
- 一時ファイルを使用する実装にする場合は、保存先と削除手順を運用手順に明記する。

## 12. エラー時挙動

主なエラーケースと対応:

| エラーケース | 検知タイミング | 利用者向け表示 | 処理方針 |
| --- | --- | --- | --- |
| 入力指定不足 | 実行前 | 必須入力不足を表示 | 変換中止 |
| 入力ファイルなし | 前処理 | ファイル不存在を表示 | 変換中止 |
| 出力先不正 | 前処理 | 出力パス不正を表示 | 変換中止 |
| フォントサイズ不正 | 前処理 | 許容範囲を表示 | 変換中止 |
| 変換失敗 | 変換中 | 変換失敗と再確認項目を表示 | 変換中止 |
| 保存失敗 | 保存時 | 書き込み不可理由を表示 | 変換中止 |

## 13. 再実行時挙動

- 既存出力ファイルがある場合:
  - 既定は上書き確認ダイアログを表示し、利用者が許可した場合のみ上書きする。
- 同一入力で再実行した場合:
  - 入力・フォント指定が同一であれば、同一内容の`.docx`を再生成する。

## 14. 非機能要件

- 可用性:
  - 外部サービスに依存しない単体実行が可能。
- 保守性:
  - Markdown要素ごとの変換処理を関数分割する。
- 操作性:
  - 職員が画面操作のみで実行できる。
- 監査性:
  - 実行結果（成功/失敗、出力先）を簡潔に表示する。

## 15. テスト観点（設計段階）

- 正常系:
  - テキスト入力→docx出力成功
  - ファイル入力→docx出力成功
  - フォント名・サイズ指定反映
- 異常系:
  - 必須入力不足
  - 入力ファイル不存在
  - フォントサイズ範囲外
  - 出力先書き込み権限不足
- 境界値:
  - 最小/最大フォントサイズ
  - 空行や長文を含むMarkdown
- 再実行:
  - 上書き拒否時の挙動
  - 上書き許可時の挙動

## 16. Office Interop 実装上の注意

- COMオブジェクト（Application / Document）は必ず解放し、Wordプロセス残留を防止する。
- Word未導入、ライセンス未認証、更新中などの状態では変換失敗し得るため、利用者向け対処を明記する。
- 変換失敗時は入力本文全体を画面表示・ログ出力しない。
- 32bit/64bit端末差異の実機確認を行い、既知制約として記録する。
