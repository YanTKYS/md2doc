# 運用引継ぎ書

## 1. 通常運用手順

1. アプリを起動する。
2. 入力方式を選ぶ。
3. Markdown入力または入力ファイルを指定する。
4. 出力先、フォント名、フォントサイズを指定する。
5. 変換実行し、結果表示を確認する。
6. 出力`.docx`内容を目視確認する。

## 2. 定期作業

- 月次でOffice更新状況を確認する。
- 変換失敗報告の傾向を確認する。
- 利用部署からの改善要望を集約する。

## 3. 更新作業

- 変更前に現行版をバックアップする。
- 改修後はテストシナリオを更新し、結果を記録する。
- 配布版には版番号を付与する。

## 4. 障害時の確認事項

- Word起動可否（手動起動できるか）。
- Officeライセンス状態。
- 出力先フォルダの書き込み権限。
- 入力データの形式妥当性。

## 5. 連絡先

- 一次対応: 運用担当部署
- 二次対応: DX担当
- 業務判断: 所管部署

## 6. 保守範囲

- DX担当: 実装修正、配布、障害解析
- 利用部署: 一次切り分け、運用調整
- 所管部署判断: 文書内容の業務責任
- 対象外: Office製品自体の不具合修正
