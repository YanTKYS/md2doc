# 利用者向け手順書

## 1. 概要

md2doc は Markdown 文書を Word ファイル（`.docx`）へ変換するツールです。  
通常は **Open XML 方式**（Word 不要・高速）で変換します。

## 2. 基本手順

1. アプリを起動する。
2. 入力方式を選ぶ（テキスト入力 / ファイル入力）。
3. Markdown 本文を入力するか、ファイルを指定する。
4. 出力先ファイル（`.docx`）を確認・指定する。
   - ファイル入力の場合は、入力ファイルと同じ場所に自動提案されます。
5. フォント名・フォントサイズを設定する。
   - 設定したフォントは本文・見出し両方のフォント種別に反映されます。
   - 見出しのフォントサイズは固定です（H1=16pt / H2=14pt / H3=12pt）。
6. 必要に応じてオプションを設定する（見出し番号・ヘッダー・フッター等）。
7. 「変換エンジン」が **Open XML 方式** になっていることを確認する。
8. `変換実行` を押す。
9. 変換完了後、Word で出力ファイルを開いて内容を目視確認する。

## 3. 設定サンプル表示の使い方

オプション設定（ヘッダー・見出し番号・ページ番号）が意図通りかを変換前に確認できます。

1. 「ヘッダー」「見出し」「フッター」などのオプションを設定する。
2. 「設定サンプル表示」欄の「サンプル更新」ボタンを押す。
3. テキストエリアに現在の設定内容が表示される。

**表示例**

```
【ヘッダー】
左：業務手順書
中央：（なし）
右：（なし）

【見出し番号】
1. 大見出し
1.1 中見出し
1.1.1 小見出し

【ページ番号】
1 / 5
```

> ※この表示は設定確認用のサンプルです。実際のWord出力とは完全には一致しません。

## 4. 変換エンジンについて

アプリ起動時は **Open XML 方式** が選択されています。通常はこのまま使用してください。

| エンジン | 説明 |
|---------|------|
| Open XML方式（標準） | Word 不要・体感 0.1 秒程度で完了 |
| Word COM方式（互換確認用） | Microsoft Word が必要・約 10 秒 |

Word COM 方式は、Open XML 方式との出力比較・互換確認が必要な場合のみ使用してください。  
なお、変換エンジンによって出力内容（順序付きリストの扱い等）に差が出る場合があります。

## 5. エラー時の基本対応

- 入力内容が空でないか確認する。
- ファイル入力の場合、ファイルのパスが正しいか確認する。
- 出力先フォルダに保存権限があるか確認する。
- Word COM 方式を使用している場合、Microsoft Word がインストール済みか確認する。
- 問題が解決しない場合は運用担当部署へ連絡する。

## 6. 注意事項

- 変換後の文書内容は必ず目視確認する。
- 個人情報を含む文書は組織ルールに従って管理する。
- 変換結果に体裁の差異がある場合、業務上許容可能かを利用部署で判断する。

## 7. 問い合わせ時に伝える内容

- いつ発生したか（日時）
- 入力方式（テキスト入力 / ファイル入力）
- 変換エンジン（Open XML 方式 / Word COM 方式）
- 表示されたエラーメッセージ
- 出力先フォルダの場所
