# 12_office_interop_checklist.md - Office Interopツール確認チェックリスト

## 1. 目的

- Office Interopを利用するC#ツールで、環境差異やCOM例外によるトラブルを減らす。
- 実機確認が必要な項目を明確にする。
- 管理者、運用担当部署、利用者それぞれの確認範囲を整理する。

## 2. 前提確認

- [ ] 利用端末に対象Office製品がインストールされている
- [ ] Officeのエディションを確認した
- [ ] Officeのバージョンを確認した
- [ ] Officeのビット数を確認した
- [ ] ライセンス上、当該利用が問題ないか確認した
- [ ] 利用者権限でOfficeを起動できる
- [ ] 初回起動時のOffice設定画面が表示されない状態で確認した
- [ ] 対象ファイル形式を開けることを確認した

## 3. C# / .NET 確認

- [ ] 対象.NETバージョンを確認した
- [ ] `dotnet build` 相当の確認を行った
- [ ] ビルド未確認の場合は理由を記録した
- [ ] 対象端末で実行確認を行った
- [ ] exe配布時の実行方法をREADMEに記載した

## 4. Office Interop 実行確認

- [ ] Officeアプリの起動に成功する
- [ ] Officeアプリの終了処理が行われる
- [ ] 変換後にOfficeプロセスが残らない
- [ ] 例外発生時にも終了処理が行われる
- [ ] COMオブジェクト解放を考慮している
- [ ] 複数回連続実行してもプロセスが残らない
- [ ] 入力ファイルが不正な場合にエラー表示される
- [ ] 出力先が存在しない場合にエラー表示される
- [ ] 既存ファイルがある場合に上書き確認される

## 5. COM例外時の対応

- 起動失敗時のエラー表示
- 変換失敗時のエラー表示
- 保存失敗時のエラー表示
- 再試行するかどうかの判断
- 自動再試行する場合の回数上限
- 自動再試行しない場合の利用者向け案内
- Word / Excel プロセス残留時の確認手順
- 手動終了が必要な場合の案内
- 管理者またはDX担当へ相談する条件

## 6. セキュリティ・個人情報

- [ ] 入力本文や変換対象文書をログ保存していない
- [ ] 外部通信していない
- [ ] 一時ファイルの保存場所を明記した
- [ ] 一時ファイルの削除方針を明記した
- [ ] 出力ファイルの保存先を利用者が確認する
- [ ] 個人情報を含む文書を扱う場合の注意をREADMEに記載した

## 7. 運用引継ぎ

- [ ] 管理者向け手順書にOffice依存を記載した
- [ ] 運用担当部署向け手順書に一次対応を記載した
- [ ] 利用者向け手順書にWord / Excel起動失敗時の案内を記載した
- [ ] 障害時に確認すべき事項を整理した
- [ ] 保守範囲と保守対象外を明記した
