# md2doc — Markdown to Word 変換ツール

## 1. 概要

Markdown形式のテキスト、またはMarkdownファイル（`.md` / `.txt`）を  
Microsoft Word形式（`.docx`）へ変換する WinForms アプリケーションです。

**Open XML方式（標準）** は Microsoft Word を起動せずに変換を完了します。  
Word COM方式（互換確認用）も引き続き選択できます。

## 2. 目的

- 一般職員が、Markdown文書を業務配布しやすいWord文書へ変換できるようにする。
- 出力文書のフォント名・フォントサイズを指定し、組織内ルールに合わせる。
- Microsoft Word が不要な環境でも変換を完結させる。

## 3. 対象利用者

- 一般職員
- 運用担当部署

## 4. 利用環境

| 条件 | Open XML方式（標準） | Word COM方式（互換確認用） |
|------|-----------------|----------------------|
| OS | Windows | Windows |
| Microsoft Office | 不要 | Microsoft Word 必須 |
| .NET 8 Runtime | 必要 | 必要 |
| 閉域ネットワーク | 対応（外部通信なし） | 対応（外部通信なし） |

## 5. 変換エンジン

アプリ起動時は **Open XML方式** が初期選択されています。  
画面下部のオプション欄から「変換エンジン」を切り替えられます。

| エンジン | 説明 | 推奨用途 |
|---------|------|---------|
| **Open XML方式**（標準） | Word 不要・高速（体感 0.1 秒） | 通常の変換作業全般 |
| Word COM方式（互換確認用） | Word 必須・約 10 秒 | Open XML出力との比較・互換確認 |

> **注意**: 変換エンジンによって出力内容に差が生じる場合があります。  
> 通常は Open XML方式で変換し、出力を Word で目視確認してください。

## 6. 対応 Markdown 要素

| 要素 | 記法 | 状態 |
|------|------|------|
| 見出し H1〜H3 | `# H1` / `## H2` / `### H3` | ✅ 対応 |
| 見出し自動番号 | オプションで付与（例: `1. / 1.1 / 1.1.1`） | ✅ 対応 |
| 通常段落 | 空行区切りのテキスト | ✅ 対応 |
| 箇条書き | `- 項目` / `* 項目` | ✅ 対応 |
| 順序付きリスト | `1. 項目` | ✅ 対応（Open XML方式のみ） |
| 表 | GFM テーブル記法 | ✅ 対応 |
| 改ページ | `<!-- pagebreak -->` / `---pagebreak---` | ✅ 対応 |
| 太字 | `**テキスト**` / `__テキスト__` | ✅ 対応 |
| 斜体 | `*テキスト*` / `_テキスト_` | ✅ 対応（Open XML方式のみ `_` 形式） |
| インラインコード | `` `コード` `` | ✅ 対応 |
| ヘッダー | オプション設定（ファイル名 / 自由記入） | ✅ 対応 |
| フッター・ページ番号 | オプション設定 | ✅ 対応 |
| 水平線 | `---`（単独行・前後に空行） | ✅ 対応 |

## 7. 既知の制約

- **画像**: 未対応。画像記法は無視されます。
- **脚注**: 未対応。
- **引用ブロック** (`>`): 未対応。
- **見出しフォントサイズの個別指定**: 未対応。H1=16pt / H2=14pt / H3=12pt 固定。
- **H4 以上の見出し**: H3 スタイルとして出力されます（番号は付与されません）。
- **順序付きリスト**: Word COM方式では通常段落として出力されます。

## 8. フォント設定

アプリの「文書フォント設定」で指定したフォントは、**本文および見出しのフォント種別** に反映されます。

| 対象 | フォント種別 | フォントサイズ |
|------|------------|-------------|
| 本文 | 文書フォント設定の指定値 | 文書フォント設定の指定値 |
| 見出し H1 | 文書フォント設定の指定値 | 16pt（固定） |
| 見出し H2 | 文書フォント設定の指定値 | 14pt（固定） |
| 見出し H3 | 文書フォント設定の指定値 | 12pt（固定） |

## 9. 使い方（基本手順）

1. アプリを起動する。
2. 入力方式（テキスト入力 / ファイル入力）を選択する。
3. Markdown本文を入力するか、入力ファイルを指定する。
4. 出力ファイル（`.docx`）を指定する（ファイル入力時は自動提案されます）。
5. フォント名・フォントサイズ、各種オプションを設定する。
6. `変換実行` を押す。
7. 変換完了後、「Wordファイルを開く」ボタンで出力内容を目視確認する。

## 10. 個人情報の取扱い

- 入力内容を外部送信しない。
- 個人情報を永続ログとして保存しない。
- 出力ファイルの保管・配布は利用部署の運用ルールに従う。

## 11. 動作確認

```bash
dotnet build src/md2doc/md2doc.csproj
dotnet test tests/Md2Doc.Tests/Md2Doc.Tests.csproj
```

- Open XML 方式のテスト（26件）は Word 不要で実行可能。
- Word COM 方式の統合テストは Windows + Microsoft Word が必要。
- `dotnet` がない環境では、Windows開発端末で確認する。

## 13. プロジェクト構成

v0.5.7 で変換処理と画面処理の責務を分離しました。

| プロジェクト | 役割 | TargetFramework |
|------------|------|----------------|
| `src/Md2Doc.Core/` | Open XML 方式の変換処理（ライブラリ） | `net8.0` |
| `src/md2doc/` | WinForms 本体（画面・設定・呼び出し） | `net8.0-windows` |
| `tests/Md2Doc.Tests/` | xUnit テストプロジェクト | `net8.0-windows` |

ソリューションファイル `md2doc.sln` で 3 プロジェクトを一括ビルドできます。

```bash
dotnet build md2doc.sln
dotnet test tests/Md2Doc.Tests/Md2Doc.Tests.csproj
```
